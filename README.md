# Terminal (mini-shell на C++)

Небольшой терминал на C++, который запускает внешние команды, поддерживает фоновые процессы и несколько встроенных команд.

## Возможности

- запуск внешних программ через `fork + execvp`
- запуск процессов в фоне через `&`
- обработка завершения фоновых процессов (`SIGCHLD`)
- список фоновых задач (`mps`)
- принудительное завершение фоновой задачи по номеру (`exit <number>`)
- смена директории (`cd`)
- выход из терминала (`close`)
- цветной prompt с `user@host` и текущим путём

## Структура проекта

- `terminal.cc` — исходный код терминала
- `Makefile` — сборка, форматирование и установка

## Требования

- Linux/Unix-подобная система
- `g++`
- `make`
- (опционально) `clang-format`
- (для установки в `/bin`) права `sudo`

## Сборка

Быстрая сборка без установки:

```bash
make terminal
```

или напрямую:

```bash
g++ -Wall -Werror -Wextra terminal.cc -o terminal
```

## Запуск

```bash
./terminal
```

## Встроенные команды

- `cd [path]` — перейти в директорию (без аргумента: в `$HOME`)
- `mps` — показать запущенные фоновые процессы
- `exit <number>` — завершить фоновый процесс по номеру из `mps`
- `chsh` — выполнить `chsh -s /bin/bash`
- `close` — закрыть терминал

Любую внешнюю команду можно запустить в фоне, добавив `&` в конец:

```bash
sleep 30 &
```

## Makefile

Цель `all` запускает:

1. `clang` (форматирование `*.cc`)
2. `clean`
3. сборку `terminal`
4. `install` (копирование в `/bin/terminal` через `sudo`)

Это значит, что `make` по умолчанию может потребовать пароль `sudo` и установленный `clang-format`.

## Примечания

- Проект ориентирован на Linux (`fork`, `waitpid`, `SIGCHLD`, `chdir`, `execvp`).
- Аргументы команд парсятся с базовой поддержкой двойных кавычек и экранирования.
